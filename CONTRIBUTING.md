# 🤝 Contributing to DeshSafe

Welcome! We're glad you're here. DeshSafe is part of **SSoC Season 5** and we'd love your help building a disaster reporting platform for India.

---

## 📋 Before You Start

- Read the [README.md](README.md) to understand the project
- Check [open issues](../../issues) before starting work
- Comment on an issue saying "I'd like to work on this" — wait for assignment before submitting a PR

---

## 🛠️ Setting Up Locally

```bash
git clone https://github.com/Anushka-045/deshsafe.git
cd deshsafe
npm install
npm run dev
```

---

## 🌿 Branching Convention

| Type | Branch Format | Example |
|------|-------------|---------|
| Feature | `feature/short-name` | `feature/report-form` |
| Bug Fix | `fix/short-name` | `fix/map-loading` |
| Docs | `docs/short-name` | `docs/update-readme` |

---

## 📝 Commit Message Format

Use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add incident report form
fix: resolve map marker overlap
docs: update contributing guide
style: fix button alignment on mobile
```

---

## 🔁 Pull Request Checklist

Before submitting a PR, make sure:

- [ ] Your code runs without errors
- [ ] You've tested your changes
- [ ] Your branch is up to date with `main`
- [ ] PR description explains what and why

---

## 🏷️ Issue Labels

| Label | Meaning |
|-------|---------|
| `good first issue` | Great for beginners |
| `help wanted` | We need contributors |
| `bug` | Something is broken |
| `enhancement` | New feature request |
| `documentation` | Docs improvements |

---

## 💬 Need Help?

Open a [Discussion](../../discussions) or comment on the relevant issue. The project admin will get back to you!
